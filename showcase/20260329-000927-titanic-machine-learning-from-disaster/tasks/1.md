在对泰坦尼克号（Titanic）数据集进行探索性数据分析（EDA）和初始数据清洗后，以下是针对数据质量、缺失值处理策略及相关性的详细总结。

### 1. 数据质量摘要 (Data Quality Summary)

数据集共包含 891 条记录。通过初步检查，识别出以下三个特征存在缺失值：

| 特征 (Feature) | 缺失值数量 (Missing Values) | 缺失比例 (Percentage) |
| :--- | :--- | :--- |
| **Age** (年龄) | 177 | 19.87% |
| **Cabin** (客舱号) | 687 | 77.10% |
| **Embarked** (登船港口) | 2 | 0.22% |

### 2. 缺失值处理策略 (Imputation or Removal Strategies)

根据各特征的缺失程度和业务逻辑，建议采取以下策略：

*   **Age (年龄)**: 
    *   **策略**：使用**中位数 (Median)** 进行填充（中位数为 28.0）。
    *   **理由**：缺失比例约为 20%，不算太高，且中位数对异常值（Outliers）不敏感，能够较好地保留数据分布特征。
*   **Cabin (客舱号)**: 
    *   **策略**：由于缺失率极高（>77%），直接填充不现实。建议**转换为二元特征 `HasCabin`**（1 代表有客舱号，0 代表缺失），随后可删除原始 `Cabin` 列。
    *   **发现**：通过分析发现，拥有客舱信息的乘客存活率（66.7%）远高于缺失信息的乘客（30.0%），说明该特征含有重要的生存信号。
*   **Embarked (登船港口)**: 
    *   **策略**：使用**众数 (Mode)** 进行填充（众数为 'S'，即 Southampton）。
    *   **理由**：仅缺失 2 条数据，使用最常见的港口填充对模型影响微乎其微。

### 3. 基本相关性分析 (Basic Correlations)

数值型特征与生存目标变量 (`Survived`) 的相关性排序如下：

| 特征 | 相关系数 (Pearson Correlation) | 解释 |
| :--- | :--- | :--- |
| **Pclass** | -0.338 | **负相关**：客舱等级越低（数字越大），生存率越低。这是最强的预测因子之一。 |
| **Fare** | 0.257 | **正相关**：票价越高，生存几率越高。 |
| **Parch** | 0.082 | **弱正相关**：父母/子女在船上的数量对生存有微弱正影响。 |
| **Age** | -0.077 | **弱负相关**：年龄越大，存活概率略微下降。 |
| **SibSp** | -0.035 | **极弱负相关**：兄弟姐妹/配偶数量的影响不明显。 |

### 4. 生成的分析文件
- `data_quality_summary.csv`: 记录了各列缺失值及比例的详细表格。
- `correlations_summary.csv`: 记录了各数值型变量与生存率的相关性系数。

**结论**：数据清洗的重点应放在 `Age` 的中位数填充和 `Cabin` 的特征工程转换上。此外，强相关性指标（如 `Pclass` 和 `Fare`）在后续建模中应作为核心特征。
