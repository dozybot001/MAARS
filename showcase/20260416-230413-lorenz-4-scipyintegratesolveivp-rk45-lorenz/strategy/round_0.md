这是一份针对 Lorenz 吸引子计算可视化研究的技术策略文档：

### 1. 关键洞察
*   **向量化即性能**：在 Python 环境下，避免 `for` 循环遍历参数空间。通过将 $N$ 个参数点包装成一个 $3N$ 维的大系统（State Vectorization），利用 NumPy 的底层的 C/Fortran 优化，可在单 CPU 核心上实现百倍以上的加速。
*   **事件驱动的精确性**：分叉图的质量取决于局部极大值的捕捉。传统的采样法（找数组最大值）受限于步长精度；利用 `solve_ivp` 的 `events` 机制结合 `direction=-1` 捕捉 $\dot{z}=0$ 的过零点，可以获得解析级的坐标精度，显著提升分叉图的锐度。
*   **MLE 饱和陷阱**：Lyapunov 指数的本质是初始微扰的指数增长。由于吸引子是有界的，两条轨迹的距离 $d(t)$ 在达到吸引子直径（Lorenz 约为 40-50）后会饱和。高性能方案必须在 $d(t)$ 进入非线性区（如 $d > 0.1$）之前截断数据，否则会因饱和导致严重低估。

### 2. 推荐方案
*   **大规模参数扫描 (Heatmap)**：
    *   **技术**：构建全参数向量化方程。定义 $Y = [x_1..x_n, y_1..y_n, z_1..z_n]$，在导数函数中使用 `dx = sigma * (y - x)` 等数组运算。
    *   **理由**：$30 \times 30$ 的网格若逐点求解约需 2-5 分钟，而向量化求解仅需约 10-20 秒，极大节省沙箱配额。
*   **分叉图生成 (Bifurcation)**：
    *   **技术**：定义事件函数 $f(t, Y) = x y - \beta z$。设置 `event.direction = -1` 并通过 `dense_output=True` 或 `events` 返回值获取精确点。
    *   **理由**：这种方法能自动跳过平滑段，仅记录峰值，且不依赖于全局高频采样，降低内存占用。
*   **MLE 定量估算**：
    *   **技术**：采用“动态截断线性拟合”。设置 $d_0 = 10^{-10}$，在积分过程中记录 $\ln(d/d_0)$。拟合区间限定在 $d(t) \in [10 d_0, 1.0]$。
    *   **理由**：无需复杂的 Benettin 重新正交化循环，在 $T=50$ 的短期仿真内，单次拟合足以获得 2 位有效数字的稳健估计（目标值 $\approx 0.91$）。

### 3. 需避免的陷阱
*   **忽视瞬态 (Transients)**：计算前必须预运行 $T_{trans} \ge 20$ 并丢弃数据。若从随机初值直接开始统计，分叉图会出现大量“噪声点”，MLE 会被非吸引子区域的动力学污染。
*   **内存溢出**：在进行大参数扫描时，严禁使用 `solve_ivp` 默认的全部轨迹存储。应仅存储 $T_{end}$ 的状态或通过 `events` 存储少量关键点。
*   **刚性计算开销**：当 $\rho > 100$ 时，系统可能出现刚性。在 CPU 限制下，优先选用 `RK45`；若报错或极慢，应切换至 `LSODA`。

### 4. 目标指标
*   **MLE 精度**：$\rho=28$ 时，$\lambda_{max}$ 应在 $[0.88, 0.93]$ 区间（理论值 $\approx 0.906$）。
*   **执行耗时**：单次 $30 \times 30$ 热力图计算应控制在 $120s$ 以内。
*   **分叉图分辨率**：$\rho$ 步长应达到 $0.1$ 且过渡区无明显断裂。
