探索性数据分析（EDA）已完成，以下是针对 `train.csv` 和 `data_description.txt` 的分析结果及相应的预处理策略。

### 1. 数据概况与缺失值分析

数据集包含 1460 条记录，共 81 个特征（含目标变量 `SalePrice`）。

**缺失值统计（主要部分）：**
*   **极高缺失率 (>80%)**: `PoolQC` (99.5%), `MiscFeature` (96.3%), `Alley` (93.8%), `Fence` (80.8%)。这些特征大多表示“无该项设施”（如无泳池、无小巷），缺失并不代表数据丢失。
*   **中高缺失率 (5% - 60%)**: `MasVnrType` (59.7%), `FireplaceQu` (47.3%), `LotFrontage` (17.7%), 以及车库相关特征 (`GarageType`, `GarageYrBlt`, `GarageFinish`, `GarageQual`, `GarageCond`) 约占 5.5%。
*   **低缺失率 (<5%)**: 地下室相关特征 (`BsmtQual` 等) 约占 2.5%，`MasVnrArea` (0.5%)，以及 `Electrical` (1个缺失)。

### 2. 特征类型确定

*   **数值型特征 (Numerical, 38个)**:
    *   关键特征包括：`LotArea`, `OverallQual`, `YearBuilt`, `GrLivArea`, `TotalBsmtSF`, `FullBath`, `GarageCars`, `SalePrice` 等。
    *   部分数值特征实际为类别或时间属性：`MSSubClass`（房屋类型编码）, `MoSold`（销售月份）, `YrSold`（销售年份）。
*   **类别型特征 (Categorical, 43个)**:
    *   包含等级型（Ordinal）：`ExterQual`, `BsmtQual`, `HeatingQC`, `KitchenQual`（评分为 Ex, Gd, TA, Fa, Po）。
    *   包含纯类别型（Nominal）：`Neighborhood`, `MSZoning`, `SaleCondition` 等。

### 3. 目标变量 `SalePrice` 分析

*   **统计分布**: 均值约 18.09万，中位数 16.3万。
*   **偏度 (Skewness)**: **1.88**（显著右偏）。
*   **峰度 (Kurtosis)**: **6.54**（长尾分布）。
*   **结论**: `SalePrice` 呈现偏态分布，建议在训练模型前进行 **Log 对数变换** 以使其更接近正态分布。

### 4. 特征偏度分析

许多数值型特征表现出高度偏性（偏度 > 0.75）：
*   极高偏度：`MiscVal` (24.45), `PoolArea` (14.81), `LotArea` (12.19)。
*   显著偏度：`GrLivArea` (1.36), `1stFlrSF` (1.37), `TotalBsmtSF` (1.52)。
*   **结论**: 对这些偏度较高的特征，建议进行 Box-Cox 变换或 Log 变换以提高模型性能。

### 5. 预处理策略 (Preprocessing Strategy)

基于上述发现，建议执行以下预处理流程：

#### A. 缺失值处理
1.  **分类变量 (NA 代表“无”)**: 对 `PoolQC`, `MiscFeature`, `Alley`, `Fence`, `FireplaceQu`, `GarageType`, `BsmtQual` 等填充为字符串 `"None"`。
2.  **分类变量 (常规缺失)**: 对 `MSZoning`, `Electrical`, `KitchenQual` 等填充为众数 (Mode)。
3.  **数值变量**:
    *   `LotFrontage`: 根据所属街区 (`Neighborhood`) 的中位数填充。
    *   `GarageYrBlt`: 填充为 0 或 `YearBuilt`。
    *   `MasVnrArea`, `BsmtFinSF1`, `TotalBsmtSF` 等地下室/车库面积：填充为 0。

#### B. 特征工程与变换
1.  **目标变量**: 使用 `np.log1p()` 变换 `SalePrice`。
2.  **数值特征**: 对偏度绝对值 > 0.75 的特征应用 `log1p` 变换。
3.  **类型转换**: 将 `MSSubClass`, `MoSold`, `YrSold` 从数值型转换为字符串/类别型。
4.  **特征组合**:
    *   `TotalSF = TotalBsmtSF + 1stFlrSF + 2ndFlrSF` (总居住面积)。
    *   `HouseAge = YrSold - YearBuilt` (房龄)。
    *   `IsNew = 1 if YearBuilt == YrSold else 0`。

#### C. 编码 (Encoding)
1.  **有序编码 (Ordinal Encoding)**: 对评价类特征（如 `ExterQual`, `KitchenQual` 等）手动映射为数字 (5, 4, 3, 2, 1)。
2.  **独热编码 (One-Hot Encoding)**: 对无序类别特征（如 `Neighborhood`, `Foundation`）进行独热编码。

#### D. 异常值处理
*   通过散点图识别并移除 `GrLivArea` 极大但 `SalePrice` 极低的异常样本（如 `GrLivArea > 4000`）。
