研究主题：合法迁移学习是否会削弱后门水印的所有权验证能力？

本项目将作为最终 showcase 案例展示，因此目标不仅是完成实验，更是产出一个结论清晰、图表直观、容易讲清楚的研究故事。核心问题是：当模型所有者为了适配新任务，对已嵌入触发器型后门水印的模型进行合法微调时，原先的所有权验证触发机制是否仍然有效。

展示导向目标：
- 产出一个可以用一句话概括的明确结论，而不是分散的实验观察。
- 优先形成适合展示的 punchline，例如“轻量迁移会部分保留水印，但全量微调会显著削弱水印”，或者与之相反的结论。
- 即使结果是否定性的，也同样有价值；关键是结论要清楚、可复现、可解释。

实验范围：
- 使用 Python 与 PyTorch 开展研究。
- 优先复现一种经典的 trigger-based watermark / backdoor watermark 方法，如 BadNets 风格触发器水印。
- 如果资源允许，可加入第二种经典方法作为补充对照，但不要无限扩展方法数量。
- 优先使用 `ResNet18` 作为主模型；如资源允许，可增加 `VGG11` 作为补充对照。
- 聚焦两类迁移场景：
  1. 将在 `MNIST` 上训练并植入水印的模型迁移到 `Fashion-MNIST`。
  2. 将在 `CIFAR10` 上训练并植入水印的模型迁移到 `CIFAR100`。
- 对每个场景至少比较两种迁移策略：
  1. 仅微调分类头（head-only / linear probing）。
  2. 全量微调（full fine-tuning）。

评估重点：
- 源任务准确率。
- 目标任务准确率。
- 迁移前后的水印触发成功率（attack success rate / ownership verification rate）。
- 水印保留率，即迁移后触发成功率相对于迁移前的变化幅度。
- 不同迁移策略下，目标任务性能与水印持久性之间的权衡。

希望回答的关键问题：
1. 现有后门水印在合法迁移学习后是否仍可用于所有权验证？
2. 哪类迁移设置最容易削弱或破坏水印？
3. 是数据集分布变化、微调方式，还是模型结构，对水印持久性的影响更大？

最终输出要求：
- 可复现实验代码、配置与运行说明。
- 一张紧凑的结果总表，汇总主要模型、迁移设置和核心指标。
- 三张最关键的展示图：
  1. 迁移前后水印触发成功率对比图。
  2. `head-only` 与 `full fine-tuning` 的对比图。
  3. 目标任务性能与水印保真度之间的权衡图。
- 一段清晰的结论分析，能够直接回答上述关键问题。
- 一个一句话 headline 和 3 到 5 条简洁 takeaway，适合最终案例展示。

研究约束：
- 本项目以实验验证和实证分析为主，不要求设计新的水印算法。
- 优先保证实验范围清晰、变量可控、结果可复现，不要为了“更全面”而显著扩大范围。
- 方法数量、模型数量和对照数量应服务于结论表达，而不是增加噪声。
- 最终应给出对模型部署、迁移和再利用全生命周期中水印有效性的明确判断。
